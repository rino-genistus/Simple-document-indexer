# Simple-document-indexer
Built this simple document indexer to be able to find things on my own computer

This simple version accesses a directory based on what you want, I hard coded this for my own computer but it will work with an input statement as well. It then goes through all the files in that directory, isolates the text files and allows you to search through them. The algorithm is a vector comparison algorithm that works by cleaning all the text file content and file paths, removing the stop words, and splitting them into tokens. The algorithm then compares the search query and the document file names and file contents using dot products and normalizing the results to find the best match between the search and files. It then returns a number between 0 to 1 showing how similar the content between the search and the returned files are. Something that I added was to prioritize the file names more than the content matches, I multiplied the file names by a constant in order to give them more priority when returning the matches.
